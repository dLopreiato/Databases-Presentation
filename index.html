Android Application Architecture

Activity

Provides a screen with which users can interact in order to do something

An application usually has multiple of these, but only 1 "main" activity. Each one is placed on the "back stack". Each one should be designed around a specific kind of action.

Task

Collection of Activities
Saved Back Stack

Home button doesn't destroy backstack. Resume a task with the "Overview Screen". Task is not an app.

Intent

Start an Activity

Start a Service

Broadcast a Message

This is how you can navigate around your own app. Might be thinking: "create intent to start my own activity? too much work. I'll just clear the screen and redraw what I want." Bad coding (no modules), breaks Android UX. (Also, good luck dealing with multiple screen sizes)
Every app receives a broadcast. Example messages: system boot up, device charging, timezone change.

Components of an Intent

  • Component Name (Optional)
  • Category
  • Action
  • Data
  • Extras
  • Flags

Real power of Intents lays here.
  • Specifying a component name makes the intent explicit.
  • Category defines what type of app can open it. (Gallery, Maps, Market, Contacts, email).
  • Action defines the exact thing the activity should do when it receives it. (View, send, edit)
  • Data is a URI that references the data the action is occurring to.
  • Extras is how you pass data long.
  • Flags do stuff specific to how the OS should handle it.

Services

Does background processing

Music Playing
Network Transactions

Works in the shadows. Can be bound (acting like server/client), or work indefinitely.

Adapters

@Override

    ...

    // fill data
    footballGamePlaylist = new ArrayList<String>();
    for (int i = 0; i < 50; i++) {
        footballGamePlaylist.add("We Dem Boyz");
    }

    // prepare adapter
    ArrayAdapter<String> adapter;
    adapter = new ArrayAdapter<String>(this, android.R.layout.simple_list_item_1, footballGamePlaylist);

    // connect adapter
    listView.setAdapter(adapter);
}
This just creates a list of strings. To create your own list, make a custom layout, and extend the adapter class.

        android:allowBackup="true"

            android:name=".MainACMActivity"

        </activity>
